## ColoriMagiques - Problème déploiement Railway résolu

**Symptôme** : Le container démarre mais crash avec 502 / application failed to respond.

**Cause racine** : `railway.toml` avait `startCommand = [\"node backend/src/server.js\"]` qui **remplace** le `CMD` du Dockerfile au lieu de le compléter. Le CMD du Dockerfile fait `sh -c 'node migrations/run.js && node migrations/seed.js && node server.js'` — la chaîne complète est nécessaire pour créer les tables ET lancer le serveur. Sans elle, `server.js` est lancé sans que les tables soient créées → crash.

**Solution** : Retirer `startCommand` du `railway.toml` pour laisser le `CMD` du Dockerfile s'exécuter entièrement.
